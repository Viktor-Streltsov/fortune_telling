# Capacitor + Android (Mystic Cards)

This project uses **Vite** with `build.outDir: build` and a **relative base** (`./`) so assets load correctly in the Android WebView.

## Prerequisites

- Node.js 18+
- Android Studio (latest stable) with Android SDK
- JDK 17 (Android Studio bundles one)

## One-time setup (already done in repo)

1. `npm install` — includes `@capacitor/*` and plugins.
2. `capacitor.config.ts` — `appId`, `appName`, `webDir: 'build'`.
3. `npx cap add android` — generated `android/` project.

## Daily workflow

```bash
# 1) Build the web app into /build
npm run build

# 2) Copy web assets + update native project
npx cap sync
# or shorthand:
npm run android:sync
```

- **`cap copy`** — copies `build/` into `android/app/src/main/assets/public`.
- **`cap sync`** — copy + updates native dependencies (plugins, Gradle).

## Open in Android Studio

```bash
npm run cap:open:android
# or after sync:
npm run android:open
```

## Build APK / AAB (Play Store)

1. **Android Studio** → open the **`android/`** folder (not the repo root).
2. **Build → Generate Signed Bundle / APK**.
3. Choose **Android App Bundle (AAB)** for Google Play (recommended) or **APK** for sideloading.
4. Create or select a keystore (keep it safe; never commit).
5. **Release** build variant → Finish.

### Upload to Google Play

1. [Google Play Console](https://play.google.com/console) → Create app.
2. **Release** → **Production** (or internal testing) → upload the **AAB**.
3. Complete store listing, content rating, privacy policy, etc.

## Icons & splash

- Default launcher icons live under `android/app/src/main/res/mipmap-*`.
- Splash uses `AppTheme.NoActionBarLaunch` and `@drawable/splash` (generated by Capacitor).
- **Recommended:** install `@capacitor/assets` and use a single source image:

  ```bash
  npx @capacitor/assets generate --iconBackgroundColor=0e0a18 --splashBackgroundColor=0e0a18
  ```

  Place `assets/icon.png` (1024×1024) and optionally `assets/splash.png` per the tool docs.

## localStorage

- **Capacitor WebView** uses the same **Web Storage API** as Chrome; `localStorage` for themes/locale/intro flags works as on the web.
- For **structured or sensitive** data later, consider **`@capacitor/preferences`** (Preferences API) instead of rolling your own.

## Troubleshooting

- **White screen / 404 assets:** ensure `vite.config.ts` has `base: './'` and you ran `npm run build` before `cap sync`.
- **Plugin not applied:** run `npx cap sync` after changing `capacitor.config.ts` or adding plugins.
- **Status bar / safe area:** CSS `env(safe-area-inset-*)` is used; `MainActivity` sets edge-to-edge via `WindowCompat.setDecorFitsSystemWindows`.
